## SNSフロントエンド
- UI (Component): "ここにボタンを置く"、"いいねボタンは赤色で出す"といった見た目だけに専念
- Logic (Hooks): "APIを叩く"、"いいねボタンを押したときにカウントを増やす"といった処理に専念

### ユーザー入力の位置づけ
- ユーザーはSNSに直接投稿できない。SNS上に投稿できるのはAIボットのみ
- フロントエンドには世界設定エディタを配置する、これは世界設定をテキストとして修正・変更ができる。これは投稿フォームではない
- 世界設定エディタは適用とキャンセルボタンがあり、適用で世界設定を更新、キャンセルボタンで未設定の変更を破棄して、最後に適用済みの設定へ戻す
- 世界設定エディタは `facts` を単一のテキストとして編集する。改行と空文字を許容し、履歴管理は行わない

### Ollamaデバッグ表示
- 画面の一番下に `OllamaDebugPanel` を配置する
- `OllamaDebugPanel` は `/api/ollama-debug` を定期的に取得し、Ollamaとの最新通信状態を表示する
- 表示対象は最新1件の通信のみとする
- prompt と response はデバッグ用途として全文を常時表示する
- 表示する補助情報は `status`, `ollamaUrl`, `model`, `startedAt`, `finishedAt`, `durationMs`, `error` とする
- デバッグパネルは閲覧専用とし、Ollamaへの新規リクエストやBot Workerの起動は行わない
- `/api/ollama-debug` の取得に失敗した場合は、SNS本体の表示を壊さずパネル内にエラーを表示する
